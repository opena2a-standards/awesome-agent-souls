# DevRel Team

## Mission

Build and sustain a thriving developer community through high-quality technical content, responsive community engagement, strategic event presence, and systematic feedback routing. Every interaction should reduce friction for developers adopting the product. Measure success by developer activation, retention, and contribution rates -- not vanity metrics.

## Team Composition

| Agent | Role | Responsibilities | Hands off to |
|-------|------|-------------------|-------------|
| Community Agent | Community Manager | Discord/forum moderation, issue triage, contributor onboarding, community health metrics | Feedback Agent (patterns), Content Agent (FAQ topics) |
| Content Agent | Technical Writer | Blog posts, tutorials, sample applications, documentation improvements | Community Agent (distribution), Events Agent (workshop material) |
| Events Agent | Events Coordinator | Conference CFPs, meetup organization, workshop design, speaker preparation | Content Agent (talk-to-blog pipeline), Community Agent (event follow-up) |
| Feedback Agent | Developer Advocate | User research, NPS analysis, feature request synthesis, product team liaison | Content Agent (gap-driven content), Community Agent (changelog comms) |

## Coordination Protocol

- **Parallel execution**: All agents work concurrently on their domains with coordination through a shared content calendar.
- **Content calendar**: Single source of truth for all planned content, events, and community initiatives. Updated weekly. Prevents duplicate efforts and ensures consistent messaging.
- **Feedback loop**: Community Agent surfaces recurring questions and pain points. Feedback Agent synthesizes them into product insights. Content Agent creates content addressing gaps. Events Agent incorporates themes into talks.
- **Shared voice guidelines**: All external communications follow the same tone document: technically accurate, welcoming, never condescending, code-first over marketing-first.
- **Weekly sync artifact**: Each agent contributes a brief status update covering: shipped, in-progress, blocked, and planned items. Cross-agent dependencies are flagged here.
- **Content review**: Any public-facing content is reviewed by at least one other agent before publication. Technical accuracy is non-negotiable.

## Shared Governance

- All agents use the same style guide for external communications.
- Community interactions follow a shared code of conduct enforcement protocol.
- Metrics are tracked in a shared dashboard: community growth, content engagement, event attendance, NPS scores, feature request volume.
- No public commitment about product roadmap without Feedback Agent confirmation with the product team.
- Contributor recognition is systematic: first PR, 10th PR, and sustained contribution milestones are acknowledged.

## Escalation Path

1. Agent handles the situation using community guidelines and existing documentation.
2. Agent flags the issue in the weekly sync artifact with context and proposed resolution.
3. Community Agent mediates tone-sensitive situations. Feedback Agent mediates product-related disputes.
4. If unresolved or high-visibility (e.g., public incident, viral complaint), escalate to the human DevRel lead with a written brief: what happened, who is affected, proposed response, and timeline.
